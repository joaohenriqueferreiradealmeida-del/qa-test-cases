**Test Cases – Responsiveness (Non-Functional)**

**Objective:**
Validate application behavior and layout consistency across multiple screen sizes and device types.

**Scope**
These test cases cover mobile, tablet, and desktop responsiveness and layout adaptability.

**Preconditions:**
- Application is accessible
- Browser responsive mode or real devices available

---

**Test Cases**

TC-RESP-01 – Mobile layout adaptation

Steps:
1. Resize viewport to mobile resolution
2. Navigate through main features

Expected Result: Layout adapts correctly with no broken elements or overflow.

---

TC-RESP-02 – Touch interaction usability

Steps:
1. Interact with buttons and links on mobile view

Expected Result: All touch elements are usable and properly sized.

---

TC-RESP-03 – Text readability on small screens

Steps:
1. Review text size and spacing on mobile resolution

Expected Result: Text remains readable without zooming.

---

TC-RESP-04 – Image scaling across devices

Steps:
1. View images on different screen sizes

Expected Result: Images scale proportionally without distortion.
